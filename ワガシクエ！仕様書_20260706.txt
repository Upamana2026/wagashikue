# ゲーム仕様書（現状実装ベース / 2026-07-06時点）

## この文書について
- 初期設計書（`ワガシクエ！初期設計書.txt`）とは別に、実装済みコードを正として現時点の仕様を記録したもの。
- 初期設計書は削除せず残す。両者に差分がある箇所は本書の内容を優先する。
- 今後キャラクターを追加する際は、本書の「2. キャラクターデータ」を参照して `js/data.js` を編集する。

## 0. 全体構成
- 静的ファイル構成（GitHub Pages想定）。サーバーなし、全データは端末内(localStorage)。
- PWA対応: `manifest.json`（standalone表示・portrait固定・テーマ色#F4A7B9）、`sw.js`（Service Worker）あり。
- 画面遷移は `App`(js/app.js) が `.screen` 要素の表示切替で管理。画面一覧:
  `menu`(メニュー) / `subject-select`(科目選択) / `battle`(戦闘) / `stock`(ストック管理) / `gacha`(ガチャ) / `encyclopedia`(図鑑) / `quiz-manage`(クイズ管理)

## 1. 戦闘システム（js/battle.js）
- Wave制: 1戦闘 = Wave1〜3の3戦(固定)。
  - Wave1・2: 通常敵候補プールから `spawnRate` に基づく抽選（`pickByRate`）。誰も当選しなければ均等抽選でフォールバック。
  - Wave3: ストック平均Lv以上のボスが存在すれば20%の確率でボス出現、外れ or 該当ボスなしの場合は通常敵抽選。
  - 通常敵候補プール: `!isBoss && level <= 10` の敵。該当がいなければ「ボス以外全部」→ それも無ければ「全敵」にフォールバック。
  - ※設計書にあった「平均Lv-1〜+2の範囲で敵を絞る」ロジックは未実装（`level<=10`固定になっている）。
- 不正解時: 該当敵の `wrongBoost`(%) を **乗算で累積**し、そのWave中の自動攻撃ダメージに反映（`currentEnemyBoost *= (1 + wrongBoost/100)`）。複数回不正解すると倍々に上がる。
- 敵の自動攻撃: 1.5秒間隔で最前列の味方(`alliesOnField[0]`)に `Math.ceil(attack * boost)` ダメージ。
- 正解時: 場に出ている味方全員の攻撃力合計を敵に与える。
- 味方の入れ替え: 最前列が倒れると自動でストックから補充（`checkAllyDefeated`）。
- 敗北条件: `alliesOnField` と `remainingStock` の両方が空になった時点。
- 勝利報酬: 角砂糖 5〜100(ランダム)、経験値は倒した敵の `exp` 合計。
- 図鑑登録: 敵を倒した瞬間に `Storage.addEncEnemy(id)` で図鑑解放（現状は撃破時のみ。設計書の「出現しただけで図鑑登録」は未実装＝倒さないと図鑑に載らない）。
- テスト用モード: `battle.js` 冒頭の `TEST_RANDOM_SPAWN` フラグ(現在false)。trueにすると5秒間隔でランダム敵が無限出現するデバッグモードになる。**本番運用では必ずfalseにすること。**

## 2. キャラクターデータ（js/data.js）
実データは `ENEMY_DATA` / `ALLY_DATA` 配列にオブジェクトとして直接記述する方式。フォルダ名からの自動パースは行っていない（フォルダ名は人間が仕様を読み取るための命名規則であり、画像パスとステータスは手動で `data.js` に転記する）。

### 2-1. 画像フォルダ（実体）
- 敵: `敵キャラ/<名前>_<攻撃力>_<HP>_<経験値>_<不正解時上昇%>％/` 配下に `normal.png`(または`nomal.png`) / `jump.png` / `attack.png`
  - ※既存フォルダはファイル名の綴りが不統一（`normal`/`nomal`、日本語の`ノーマル/ジャンプ/アタック`混在）。新規キャラは `normal.png`/`jump.png`/`attack.png` に統一推奨。
- 味方: `味方キャラ/<名前>_<攻撃力>_<レアリティ%>_<Lv>/` 配下に同様の差分画像。jump/attackが無いキャラも許容（`images.jump`/`images.attack` を `null` にすればnormalで代用される）。

### 2-2. 敵オブジェクトの形式（ENEMY_DATA）
```js
{
  id: 'kebab-case-id',       // 一意なID。図鑑・撃破処理で使用
  name: '表示名',
  level: 数値,               // 出現Lv帯の判定に使用（フォルダ名にLv指定が無い場合は
                             //  ステータスから逆算した仮値を設定している）
  attack: 数値,
  hp: 数値,
  exp: 数値,                 // 撃破時の獲得経験値
  wrongBoost: 数値,          // 不正解時の攻撃力上昇%（例: 20 → 120%）
  spawnRate: 0〜1,           // Wave抽選での当選確率。Lv10以下・非ボスは一律0.30運用中
  isBoss: true/false,        // true の場合はWave3でのみ、条件成立時に低確率で出現
  images: { normal, jump, attack },  // 相対パス。jump/attackは無ければnormalと同じでも可
  description: '図鑑説明文（\nで改行可）'
}
```
- 現在登録済み3体: みたらし団子(Lv2, 攻5/HP15/経15/上昇20%) / 苺大福(Lv5, 攻20/HP50/経50/上昇12%) / 水ようかん(Lv6, 攻30/HP65/経20/上昇20%)
- ボス(`isBoss:true`)は現状0体（未実装・今後追加予定の項目）。

### 2-3. 味方オブジェクトの形式（ALLY_DATA）
```js
{
  id: 'kebab-case-id',
  name: '表示名',
  attack: 数値,
  rarity: 数値,        // ガチャ排出率。ALLY_DATA全体のrarity合計に対する重み抽選
  level: 数値,
  isInitial: true/false, // true のキャラは初回起動時に自動でストックへ付与される
  images: { normal, jump, attack },
  description: '図鑑説明文'
}
```
- HPは保存せず `getAllyMaxHp(data) = 50 + level * 20` で都度算出。
- 現在登録済み2体: 紅茶(攻30/レア30%/Lv5/isInitial:true) / 生ビール(攻40/レア12%/Lv6/isInitial:false)
- `isInitial:true` のキャラは既存セーブにも一度だけ後付けで補充される（`Storage.INIT_GRANTED` フラグで制御、再付与はされない）。

### 2-4. 新規キャラ追加の手順（実務フロー）
1. `敵キャラ/` または `味方キャラ/` 配下に `名前_ステータス.../` フォルダを作り、差分画像(normal/jump/attack)を入れる。
2. `js/data.js` の `ENEMY_DATA`（または `ALLY_DATA`）配列末尾にオブジェクトを1件追記する。
   - `id` は他と重複しない kebab-case。
   - `images` のパスはフォルダ名・ファイル名と完全一致させる（表記ゆれに注意）。
3. 敵の場合、`level` は「味方の平均Lv±どの範囲で出したいか」を考慮して設定（現状の運用値は仮設定が多いため、本来の狙いのLvに寄せて決めてよい）。
4. 味方の場合、`rarity` は他キャラとの相対値でガチャ排出率が決まる点に注意（合計に対する割合）。

### 2-5. ロード方式
- 設計書にあった「起動時は読み込まず入手時に遅延読み込み」は未実装。現状は `ENEMY_DATA`/`ALLY_DATA` 定義時点でパスを持つのみで、実際の画像ロードは各画面のDOM描画(`<img src>`)時に発生する（＝ブラウザの通常の画像ロード任せ。図鑑の未入手キャラは画像を出さずシルエット表示することで実質的に遅延させている）。

## 3. 図鑑機能（js/encyclopedia.js）
- タブ切替: 味方(ally) / 敵(enemy)。
- 味方: 入手済み(`Storage.getEncAllies()`に含まれる)なら画像・攻撃力・Lv・説明文を表示。未入手はシルエット「？」+「入手で解放」。
- 敵: 撃破済み(`Storage.getEncEnemies()`)なら画像・攻撃力・HP・説明文を表示。未撃破はシルエット+「倒すと解放」。
- 保存先: localStorage（`wq_encAllies` / `wq_encEnemies`）。

## 4. 音声・音楽
- BGM（js/bgm.js）: 画面が `battle` のときのみ `Samurai_Strain.mp3`、それ以外の全画面共通で `新年の風.mp3`。ループ再生。音量/ミュートは端末設定に依存させアプリ側では操作しない。自動再生ブロック時は最初のユーザー操作で再生開始。
- 効果音（js/battle.js内SE）: 味方攻撃時 `CHIPTUNE発射.mp3`、敵攻撃時 `enemy_attack.mp3`。HTMLAudioのプール(各3個)を使い回し、初回ユーザー操作でミュート再生→解除する「unlock」処理でタイマー起因の自動再生ブロックを回避。
- キャラクター個別ボイスは現状未実装（BGM/SEのみ）。

## 5. 画像の白背景透過処理（js/imageproc.js）
- 表示用の画像は読み込み時にCanvasで解析し、透明ピクセルが無ければ白背景とみなして輝度ベースで透過加工する（`_removeBg`）。
- 処理結果はURLごとにキャッシュ(`_imgCache`)し再処理しない。
- file://読み込み等でCanvasが汚染されGetImageDataが失敗する場合は何も加工せずそのまま表示（以前試みたmix-blend-mode:multiplyは白い体が背景に溶けるバグがあったため廃止）。
- 対象: バトル用スプライト(`processCharSprite`: normal/jump/attack全パターン)、静止画(`processStaticImg`: 図鑑・ガチャ結果・ストック一覧)。

## 6. プラットフォーム・配信
- 設計書の内容から変更なし。GitHub Pages想定の静的ファイル。iPhone/Android/PCブラウザ対象。対人対戦なし。
- PWAとして `manifest.json`+`sw.js` を追加済み（設計書には無い実装追加分）。

## 7. ユーザー経験値・レベル、ストック（js/storage.js, js/data.js）
- 次レベル必要経験値 = 現在レベル × 102（設計書通り）。
- ストック上限式: **設計書と異なる実装になっている点に注意**
  - 実装: Lv10未満は2体固定。Lv10以上は `Math.floor(レベル/10) + 2`（例: Lv10→3体, Lv20→4体, Lv30→5体）
  - 設計書の記述「レベルの十の位×2」（例: Lv23→4体）とは値が異なる。**本書（実装）を正とする。**
- 初期付与キャラ: `ALLY_DATA` 内 `isInitial:true` のキャラ（現状は紅茶のみ）が初回起動時・および未付与の既存セーブに対して自動でストックに追加される。
- ストック並び替え: ドラッグ&ドロップ、または▲▼ボタン。解放（削除）も可能。
- 保存キー(localStorage): `wq_userLevel` / `wq_userExp` / `wq_userSugar` / `wq_stock` / `wq_encAllies` / `wq_encEnemies` / `wq_quizSets` / `wq_initialGranted`

## 8. 角砂糖（コイン）
- 設計書通り。戦闘勝利で5〜100角砂糖獲得。保存先はlocalStorage。

## 9. ガチャ（js/gacha.js）
- 消費: 200角砂糖固定。
- 抽選方式: `ALLY_DATA` 各キャラの `rarity` を重みとした比例抽選（合計値に対するランダム値で判定。設計書通り「フォルダ名のレアリティ%」に従う実装）。
- ストック上限到達時: 「ノミモノたちが既にいっぱいです！」表示、角砂糖は消費しない。
- 角砂糖不足時: 「角砂糖が足りません！」表示（必要数・所持数を提示）。
- 排出結果が既存図鑑未登録なら「✨NEW！」、登録済みなら「💫ゲット！」表示。

## 10. クイズ問題の管理（js/quiz.js）
- 出題形式・インポート仕様（CSV/Excel、1列目問題文/2列目正解/3〜5列目誤答、1行目ヘッダ除外）は設計書通り実装済み。
- 誤答不足時のランダム補完: 同科目内の他問題の正解からランダム抽出（設計書通り）。それでも数が足りない場合は `？` で埋める（設計書に無い実装上のフォールバック）。
- デフォルト科目「九九」（2〜9の段、4択）は起動時に自動投入。UIから削除不可（削除ボタン非表示）で、既存セーブから欠落していても自動復元される。
- 科目・問題の保存先: localStorage（`wq_quizSets`）。
