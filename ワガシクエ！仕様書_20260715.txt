## 0. 全体構成
- 静的ファイル構成（GitHub Pages想定）。サーバーなし、全データは端末内(localStorage)。
- PWA対応: `manifest.json`（standalone表示・portrait固定・テーマ色#F4A7B9）、`sw.js`（Service Worker、cache-first）あり。
  - **Service Workerキャッシュバージョン: `wagashikue-v18`**（2026-07-15更新。サドンデスモード追加に伴い `Kurba.mp3` をアプリシェルに追加し、旧v17から更新）。
  - ファイルを更新したら必ず `sw.js` の `CACHE` 定数（`wagashikue-vN`）をインクリメントすること。インストール済みPWAは、ページ読み込み時に新しいService Workerがバックグラウンドでインストール→旧キャッシュを削除→次回起動時（または`skipWaiting`+`clients.claim`によりほぼ即時）に新バージョンへ切り替わる。
- 画面遷移は `App`(js/app.js) が `.screen` 要素の表示切替で管理。画面一覧:
  `menu`(メニュー) / `subject-select`(科目選択) / `battle`(戦闘) / `stock`(ストック管理) / `gacha`(ガチャ) / `encyclopedia`(図鑑) / `quiz-manage`(クイズ管理)

## 1. 戦闘システム（js/battle.js）

### 1-1. 通常モード
- Wave制: 1戦闘 = Wave1〜3の3戦(固定)。
  - Wave1・2: 通常敵候補プールから `spawnRate` に基づく抽選（`pickByRate`）。誰も当選しなければ均等抽選でフォールバック。
  - Wave3（最終Wave）: ストック平均Lv以下のボスが存在すれば20%の確率でボス出現、外れ or 該当ボスなしの場合は通常敵抽選。
  - 通常敵候補プール: `!isBoss && level <= 10` の敵。該当がいなければ「ボス以外全部」→ それも無ければ「全敵」にフォールバック。
- 不正解時: 該当敵の `wrongBoost`(%) を **乗算で累積**し、そのWave中の自動攻撃ダメージに反映（`currentEnemyBoost *= (1 + wrongBoost/100)`）。複数回不正解すると倍々に上がる。Wave が変わるとリセットされる（Waveをまたいで累積しない）。
- 敵の自動攻撃: 1.5秒間隔で最前列の味方(`alliesOnField[0]`)に `Math.ceil(attack * boost)` ダメージ。
- 正解時: 場に出ている味方全員の攻撃力合計を敵に与える。
- 味方の入れ替え: 最前列が倒れると自動でストックから補充（`checkAllyDefeated`）。
- 敗北条件: `alliesOnField` と `remainingStock` の両方が空になった時点。
- 勝利報酬: Wave3をクリアした時点で **角砂糖5〜100(ランダム、一括)** と **経験値（倒した敵の`exp`合計、一括）** を付与。
- 敗北時の報酬: **無し**（通常モードは敗北すると経験値・角砂糖ゼロ）。

### 1-2. サドンデスモード（2026-07-15追加）
- 「科目を選んでバトル」画面の右上に配置したトグルボタン（`#sudden-death-toggle`、`QuizManager.toggleSuddenDeath()`）でON/OFF切替。
  - ON状態は画面遷移中は保持されるが、**アプリの再読込・再起動では保持されない**（毎回OFFから始まる。localStorageに保存していない）。
  - 状態管理: `js/quiz.js` 内モジュール変数 `suddenDeathMode`。バトル開始時に `Battle.startBattle(subjectName, suddenDeathMode)` として渡す。
- Wave数: 通常3Waveの代わりに **最大10Wave** まで挑戦できる（`buildWaves(avgLv, waveCount)` の `waveCount` を10に指定）。
  - Wave1〜9: 通常モードのWave1・2と同じ抽選ロジック（`pickByRate`、Lv10以下・非ボスのプールから抽選）。
  - Wave10（最終Wave）: 通常モードのWave3同様、ボスが存在すれば20%抽選、それ以外は通常抽選。
  - 敵の強さ自体は通常モードと同じプールを使い回しており、Waveが進むほど強くなる調整はしていない（意図的な仕様。今後ボスキャラや高Lv帯の敵が追加されれば体感難度は自然に上がる）。
- 報酬（最大の特徴）: 敵を1体倒すたびに **その場で確定して経験値・角砂糖を付与**する（`enemyDefeated()` 内、`state.suddenDeath` 分岐）。
  - 経験値: 敵の `exp` をそのまま `Storage.addExp()` に渡す（通常モードと同じ計算式）。
  - 角砂糖: `sugarForEnemy(enemy)` = `3 + ランダム(0〜敵のexpまたは5の大きい方)`。例: exp20の敵なら3〜23個。
  - レベルアップ回数は `state.leveledTotal` に加算し続け、バトル終了時にまとめて表示する。
- **敗北しても、それまでに倒した敵の分の経験値・角砂糖は失われない**（既にStorageへ確定済みのため）。「できる限り連続して問題を解く」というモードの趣旨に合わせた設計。
- 結果画面:
  - Wave10まで勝ち抜いた場合: 「🔥 サドンデス 完全制覇！」＋「Wave 10 まで勝ち抜いた！」＋累計経験値・角砂糖・レベルアップ表示。
  - 味方全滅で途中敗北した場合: 「💧 敗北...」＋「Wave N で戦えるキャラがいなくなってしまった...」＋その時点までの累計経験値・角砂糖を表示（通常モードの「ガチャで仲間を増やそう！」文言は出さない）。
- Wave表示: 通常は `Wave X / 3`、サドンデス時は `🔥 Wave X / 10` と接頭辞を付けて区別。
- BGM: サドンデスモードでバトル画面に入ると `Kurba.mp3` をループ再生（通常バトルは `Samurai_Strain.mp3`）。`Battle.startBattle()` の冒頭で `BGM.setSuddenDeath(true/false)` を呼び、その後の `App.goTo('battle')` 経由の `BGM.playFor('battle')` が参照する再生対象を切り替える仕組み（`js/bgm.js`）。

### 1-3. テスト用モード（変更なし）
- `battle.js` 冒頭の `TEST_RANDOM_SPAWN` フラグ(現在false)。trueにすると5秒間隔でランダム敵が無限出現するデバッグモードになる。**本番運用では必ずfalseにすること。**
- サドンデスモードのフラグと共存可能だが、テストモード中はWave制のロジック自体を使わないため、サドンデス側のWave10終了判定は機能しない（デバッグ専用の組み合わせであり通常は同時に使わない）。

## 2. キャラクターデータ（js/data.js）
実データは `ENEMY_DATA` / `ALLY_DATA` 配列にオブジェクトとして直接記述する方式。フォルダ名からの自動パースは行っていない（フォルダ名は人間が仕様を読み取るための命名規則であり、画像パスとステータスは手動で `data.js` に転記する）。

### 2-1. 画像フォルダ（実体）
- 敵: `敵キャラ/<名前>_<攻撃力>_<HP>_<経験値>_<不正解時上昇%>％/` 配下に `normal.png`(または`nomal.png`) / `jump.png` / `attack.png`
  - ※既存フォルダはファイル名の綴りが不統一（`normal`/`nomal`、日本語の`ノーマル/ジャンプ/アタック`混在）。新規キャラは `normal.png`/`jump.png`/`attack.png` に統一推奨。
- 味方: `味方キャラ/<名前>_<攻撃力>_<レアリティ%>_<Lv>/` 配下に同様の差分画像。jump/attackが無いキャラも許容（`images.jump`/`images.attack` を `null` にすればnormalで代用される）。

### 2-2. 敵オブジェクトの形式（ENEMY_DATA）
```js
{
  id: 'kebab-case-id',       // 一意なID。図鑑・撃破処理で使用
  name: '表示名',
  level: 数値,               // 出現Lv帯の判定に使用（フォルダ名にLv指定が無い場合は
                             //  ステータスから逆算した仮値を設定している）
  attack: 数値,
  hp: 数値,
  exp: 数値,                 // 撃破時の獲得経験値（サドンデス時はこの値を都度即時付与）
  wrongBoost: 数値,          // 不正解時の攻撃力上昇%（例: 20 → 120%）
  spawnRate: 0〜1,           // Wave抽選での当選確率。Lv10以下・非ボスは一律0.30運用中
  isBoss: true/false,        // true の場合は最終Wave（通常3/サドンデス10）でのみ、条件成立時に低確率で出現
  images: { normal, jump, attack },  // 相対パス。jump/attackは無ければnormalと同じでも可
  description: '図鑑説明文（\nで改行可）'
}
```
- 現在登録済み3体（2026-07-15時点の実データ。旧仕様書の水ようかんの値は古いので本書の値が正）:
  - みたらし団子: Lv2, 攻5/HP15/経15/上昇20%
  - 苺大福: Lv5, 攻20/HP50/経50/上昇12%
  - 水ようかん: Lv6, 攻15/HP30/経20/上昇20%（旧仕様書では攻30/HP65と記載されていたが、その後のステータス調整で攻15/HP30に変更済み）
- ボス(`isBoss:true`)は現状0体（未実装・今後追加予定の項目）。

### 2-3. 味方オブジェクトの形式（ALLY_DATA）
```js
{
  id: 'kebab-case-id',
  name: '表示名',
  attack: 数値,
  rarity: 数値,        // ガチャ排出率。ALLY_DATA全体のrarity合計に対する重み抽選
  level: 数値,
  isInitial: true/false, // true のキャラは初回起動時に自動でストックへ付与される
  images: { normal, jump, attack },
  description: '図鑑説明文'
}
```
- HPは保存せず `getAllyMaxHp(data) = 50 + level * 20` で都度算出。
- 現在登録済み2体: 紅茶(攻30/レア30%/Lv5/isInitial:true) / 生ビール(攻40/レア12%/Lv6/isInitial:false)
- `isInitial:true` のキャラは既存セーブにも一度だけ後付けで補充される（`Storage.INIT_GRANTED` フラグで制御、再付与はされない）。

### 2-4. 新規キャラ追加の手順（実務フロー）
1. `敵キャラ/` または `味方キャラ/` 配下に `名前_ステータス.../` フォルダを作り、差分画像(normal/jump/attack)を入れる。
2. `js/data.js` の `ENEMY_DATA`（または `ALLY_DATA`）配列末尾にオブジェクトを1件追記する。
   - `id` は他と重複しない kebab-case。
   - `images` のパスはフォルダ名・ファイル名と完全一致させる（表記ゆれに注意）。
3. 敵の場合、`level` は「味方の平均Lv±どの範囲で出したいか」を考慮して設定（現状の運用値は仮設定が多いため、本来の狙いのLvに寄せて決めてよい）。サドンデスモードは同じ抽選プールをWave10まで使い回すため、追加した敵はサドンデスにもそのまま反映される。
4. 味方の場合、`rarity` は他キャラとの相対値でガチャ排出率が決まる点に注意（合計に対する割合）。
5. アプリシェルに音声・画像ファイルを追加した場合は `sw.js` の `APP_SHELL` 配列にも相対パスを追加し、`CACHE` バージョンを上げること（インストール済みPWAに反映するため必須）。

### 2-5. ロード方式
- 設計書にあった「起動時は読み込まず入手時に遅延読み込み」は未実装。現状は `ENEMY_DATA`/`ALLY_DATA` 定義時点でパスを持つのみで、実際の画像ロードは各画面のDOM描画(`<img src>`)時に発生する（＝ブラウザの通常の画像ロード任せ。図鑑の未入手キャラは画像を出さずシルエット表示することで実質的に遅延させている）。

## 3. 図鑑機能（js/encyclopedia.js）
- タブ切替: 味方(ally) / 敵(enemy)。
- 味方: 入手済み(`Storage.getEncAllies()`に含まれる)なら画像・攻撃力・Lv・説明文を表示。未入手はシルエット「？」+「入手で解放」。
- 敵: 撃破済み(`Storage.getEncEnemies()`)なら画像・攻撃力・HP・説明文を表示。未撃破はシルエット+「倒すと解放」。
- 保存先: localStorage（`wq_encAllies` / `wq_encEnemies`）。
- サドンデスモードで倒した敵も通常モードと同じ `Storage.addEncEnemy(id)` を通るため、図鑑登録の扱いに差はない。

## 4. 音声・音楽
- BGM（js/bgm.js）:
  - 画面が `battle` のとき: 通常は `Samurai_Strain.mp3`、**サドンデスモードでは `Kurba.mp3`**（`BGM.setSuddenDeath(bool)` で切替。`Battle.startBattle()` から呼ばれる）。
  - それ以外の全画面共通: `新年の風.mp3`。
  - いずれもループ再生。音量/ミュートは端末設定に依存させアプリ側では操作しない。自動再生ブロック時は最初のユーザー操作で再生開始。
- 効果音（js/battle.js内SE）: 味方攻撃時 `CHIPTUNE発射.mp3`、敵攻撃時 `enemy_attack.mp3`。HTMLAudioのプール(各3個)を使い回し、初回ユーザー操作でミュート再生→解除する「unlock」処理でタイマー起因の自動再生ブロックを回避。
- キャラクター個別ボイスは現状未実装（BGM/SEのみ）。

## 5. 画像の白背景透過処理（js/imageproc.js）
- 表示用の画像は読み込み時にCanvasで解析し、透明ピクセルが無ければ白背景とみなして輝度ベースで透過加工する（`_removeBg`）。
- 処理結果はURLごとにキャッシュ(`_imgCache`)し再処理しない。
- file://読み込み等でCanvasが汚染されGetImageDataが失敗する場合は何も加工せずそのまま表示（以前試みたmix-blend-mode:multiplyは白い体が背景に溶けるバグがあったため廃止）。
- 対象: バトル用スプライト(`processCharSprite`: normal/jump/attack全パターン)、静止画(`processStaticImg`: 図鑑・ガチャ結果・ストック一覧)。

## 6. プラットフォーム・配信
- GitHub Pages想定の静的ファイル。iPhone/Android/PCブラウザ対象。対人対戦なし。
- リポジトリ: `https://github.com/Upamana2026/wagashikue`（origin/main）。
- PWAとして `manifest.json`+`sw.js` を追加済み。既にホーム画面に追加（インストール）済みの端末へ更新を反映するには、`sw.js` の `CACHE` バージョンを上げてpush → 端末側でアプリを開く（オンライン状態でリロード）とバックグラウンドで新しいService Workerが取得され、次回起動時に新バージョンへ切り替わる。即時反映させたい場合はアプリを完全に閉じて開き直す（または一度リロードしてから再度開く）のが確実。

## 7. ユーザー経験値・レベル、ストック（js/storage.js, js/data.js）
- 次レベル必要経験値 = 現在レベル × 102。
- ストック上限式:
  - 実装: Lv10未満は2体固定。Lv10以上は `Math.floor(レベル/10) + 2`（例: Lv10→3体, Lv20→4体, Lv30→5体）
  - 上記のレベル依存式による上限（base）に加え、角砂糖500個消費でのストック拡張分（`Storage.getStockExpansion()`、永続加算・購入回数に上限なし）を加算した値が実際の上限となる。拡張はガチャ画面のボタンから行う（`js/gacha.js` の `expandStock`）。保存キー: `wq_stockExpansion`。
- 初期付与キャラ: `ALLY_DATA` 内 `isInitial:true` のキャラ（現状は紅茶のみ）が初回起動時・および未付与の既存セーブに対して自動でストックに追加される。
- ストック並び替え: ドラッグ&ドロップ、または▲▼ボタン。解放（削除）も可能。
- 保存キー(localStorage): `wq_userLevel` / `wq_userExp` / `wq_userSugar` / `wq_stock` / `wq_encAllies` / `wq_encEnemies` / `wq_quizSets` / `wq_initialGranted` / `wq_stockExpansion`
- サドンデスモードのON/OFF状態自体はlocalStorageに保存されない（画面遷移中のみ保持されるJSのメモリ上の状態）。

## 8. 角砂糖（コイン）
- 通常モード: 戦闘勝利（Wave3クリア）時に5〜100角砂糖を一括獲得。敗北時は0。
- サドンデスモード: 敵を倒すたびに `sugarForEnemy()` の式で都度加算（「1-2. サドンデスモード」参照）。途中敗北しても確定済みの分は残る。
- 保存先はlocalStorage（`wq_userSugar`）。

## 9. ガチャ（js/gacha.js）
- 消費: 200角砂糖固定。
- 抽選方式: `ALLY_DATA` 各キャラの `rarity` を重みとした比例抽選（合計値に対するランダム値で判定）。
- ストック上限到達時: 「ノミモノたちが既にいっぱいです！」表示、角砂糖は消費しない。
- 角砂糖不足時: 「角砂糖が足りません！」表示（必要数・所持数を提示）。
- 排出結果が既存図鑑未登録なら「✨NEW！」、登録済みなら「💫ゲット！」表示。
- ストック拡張ボタン: 角砂糖500個消費でストック上限を永続的に+1（`Storage.addStockExpansion`）。ガチャ画面に現在のストック数/上限表示とボタンを常設。角砂糖不足時はガチャと同様のエラー表示、成功時は緑系の成功メッセージを表示。

## 10. クイズ問題の管理（js/quiz.js）
- 出題形式・インポート仕様（CSV/Excel、1列目問題文/2列目正解/3〜5列目誤答、1行目ヘッダ除外）。
- 誤答不足時のランダム補完: 同科目内の他問題の正解からランダム抽出。それでも数が足りない場合は `？` で埋める。
- デフォルト科目「九九」（2〜9の段、4択）は起動時に自動投入。UIから削除不可（削除ボタン非表示）で、既存セーブから欠落していても自動復元される。
- 科目・問題の保存先: localStorage（`wq_quizSets`）。
- 「科目を選んでバトル」画面（`#screen-subject-select`）の右上にサドンデスモードのトグルボタンを常設（`renderSubjectSelect()` 内で `renderSuddenDeathToggle()` を呼び毎回状態を反映）。
